  rows/key max         : 800
  max/median ratio     : 800.0
  classification       : pathological-skew

  top keys:
    cust_hot                                          800  80.00%
    cust_155                                            1   0.10%
    (... remaining keys with 1 row each ...)

How to interpret
----------------
* **classification pathological-skew** — max/median ratio is extreme; expect a long-tail task on
  the next shuffle grouped by this key (Spark UI: one task with far higher shuffle read/duration).
* **Top key share (80%)** — A single value dominates; consider salting, two-phase aggregation,
  AQE skew hints (if applicable), or business-aware splitting before paying for a full shuffle.
* **distinct keys vs shuffle partitions** — Compare `distinct keys` to `spark.sql.shuffle.partitions`;
  if partitions >> keys you may be over-sharding; if keys >> partitions, watch merge pressure.
