How to interpret
----------------
* **100% under the small-file threshold** — Every file is far below a healthy Parquet target
  (often hundreds of MiB for analytics). Reads will spawn many tasks; metadata and listing costs grow.
* **Per-partition file counts** — Use this to see whether the problem is global or isolated to a few
  prefixes before choosing compaction scope.
* **Production path** — Re-run with `--path s3://...` or `file://...` on your dataset (credentials as
  required for cloud storage).
