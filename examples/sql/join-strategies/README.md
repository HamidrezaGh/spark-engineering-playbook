# Join strategy examples (Spark SQL)

Small **EXPLAIN** exercises that show **broadcast** vs **sort-merge** on the same join.

## How to run

From the repo root, use the local harness (loads `events` and `campaigns` and runs Spark SQL):

```bash
cd examples/local
./run_examples.sh sql
```

Or run a single file (still needs the init that creates the views):

```bash
spark-sql --conf spark.sql.shuffle.partitions=8 -f examples/local/.init.sql \
  -f examples/sql/join-strategies/broadcast_join.sql
```

The init file is generated by `run_examples.sh`; to run standalone, use `run_examples.sh` or
copy the `CREATE VIEW` block from the script into your session.

**Faster path:** `run_examples.sh` writes `examples/local/.init.sql` when it runs — after one full
`sql` run you can re-use that file.

## What to observe

- **`broadcast_join.sql`:** `BroadcastHashJoin` and `BroadcastExchange` in `EXPLAIN FORMATTED`.
- **`sort_merge_join.sql`:** `SortMergeJoin` with two `Exchange` and two `Sort` children after
  disabling auto-broadcast.

## Production lesson

Join strategy is chosen from **estimates and thresholds**. A silent regression is often
**stale** or **missing** statistics, not a random Spark upgrade.
